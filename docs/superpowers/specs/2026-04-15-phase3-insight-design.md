# OHCS SmartGate — Phase 3: Insight Design

**Date:** 2026-04-15
**Status:** Approved
**Author:** Claude + Ozzy

## Goal

Add analytics, reports, visit history search, and installable PWA to SmartGate. Give directors strategic insight into visitor patterns and give receptionists a comprehensive visit log. Make SmartGate installable as a standalone app on the reception desk.

## Scope

| Feature | Priority | Complexity |
|---|---|---|
| Analytics Dashboard | High | Medium |
| Reports (PDF + CSV) | High | Medium |
| Visit Log (search/filter) | High | Low |
| Installable PWA | Medium | Low |

---

## 1. Analytics Dashboard (`/analytics`)

### Audience
- **Receptionist**: glances at today's snapshot (top section)
- **Director/Superadmin**: scrolls down for trends and patterns (bottom section)

### Page Layout

#### Top Section — Today's Snapshot

6 stat cards in a responsive grid:
- Total Visitors Today
- Currently In Building
- Checked Out Today
- Average Duration (minutes)
- Peak Hour (e.g. "10:00 AM")
- Busiest Directorate (e.g. "RTDD")

Below the cards: a horizontal bar chart showing visits per directorate for today.

#### Bottom Section — Trends

Period selector tabs: **7 days** | **30 days** | **90 days**

Charts (all rendered client-side via Recharts):
1. **Daily Visit Volume** — line chart, x-axis = date, y-axis = visit count
2. **Visits by Day of Week** — bar chart (Mon-Sun), shows the average pattern
3. **Visits by Hour of Day** — bar chart (8 AM - 5 PM), shows peak hours
4. **Purpose Categories** — donut chart showing the distribution of visit purpose categories
5. **Top 10 Frequent Visitors** — table with name, organisation, visit count, last visit date

## 2. Reports (`/reports`)

### Page Layout

**Report Configuration Panel:**
- Period selector: radio buttons for "Today", "This Week", "This Month", "Custom"
- Custom date range: From/To date pickers (shown when "Custom" selected)
- Report type dropdown: "Summary Report", "Detailed Visit Log", "Directorate Breakdown"
- Two action buttons: "Download PDF" and "Export CSV"

**Preview Section:**
- Shows a summary of what the report will contain (record count, date range) before downloading

### PDF Generation (client-side)

Library: `jspdf` + `jspdf-autotable`

PDF structure:
1. **Header**: OHCS logo (embedded as base64), "OHCS SmartGate — Visitor Management Report", date range, generation timestamp
2. **Ghana flag bar**: 3px line in red/gold/green
3. **Summary stats**: Total visits, unique visitors, average duration, busiest directorate, busiest day
4. **Visit table**: Date, Visitor Name, Organisation, Host, Directorate, Purpose, Check In, Check Out, Duration, Status
5. **Footer**: "Generated by OHCS SmartGate" + page numbers

Filename: `OHCS-SmartGate-Report-YYYY-MM-DD.pdf`

### CSV Export (client-side)

Same data as the PDF table, exported as comma-separated values. Fields: Date, Visitor Name, Organisation, Host Officer, Directorate, Purpose, Check In Time, Check Out Time, Duration (min), Status, Badge Code.

Filename: `OHCS-SmartGate-Export-YYYY-MM-DD.csv`

### API Endpoint

```
GET /api/reports/visits?from=2026-04-01&to=2026-04-15&directorate_id=&limit=500
```

Returns the full visit data for the period with joined visitor/officer/directorate names. Same shape as existing visits list endpoint but with `from`/`to` date range support and a higher default limit.

### Sidebar
New nav item "Reports" with a `FileText` icon, visible to all roles. Positioned after "Analytics".

## 3. Visit Log (`/visit-log`)

### Page Layout

**Filter Bar (top of page):**
- Search input: searches visitor name, organisation, badge code
- Date range: From / To date pickers
- Status dropdown: All | Checked In | Checked Out | Cancelled
- Directorate dropdown: populated from `GET /api/directorates`
- "Clear Filters" button

**Table:**
| Column | Width |
|---|---|
| Date | narrow |
| Visitor Name | wide |
| Organisation | medium |
| Host Officer | medium |
| Directorate | narrow (badge) |
| Purpose | medium (truncated) |
| Check In | narrow |
| Check Out | narrow |
| Duration | narrow |
| Status | narrow (badge) |

**Pagination:**
- Cursor-based, "Load More" button at bottom
- Default: 50 visits per page

**Interactions:**
- Click visitor name → navigate to `/visitors/:id`
- Default view: today's visits, most recent first

### API Changes

Update existing `GET /api/visits` to support additional query params:
- `from` — start date (ISO date string)
- `to` — end date (ISO date string)
- `directorate_id` — filter by directorate
- `q` — search visitor name, organisation, or badge code (requires JOIN with visitors table)

### Sidebar
New nav item "Visit Log" with a `ScrollText` icon, visible to all roles. Positioned after "Visitors" and before "Analytics".

## 4. Installable PWA

### Manifest (`/public/manifest.json`)

```json
{
  "name": "OHCS SmartGate — Visitor Management System",
  "short_name": "SmartGate",
  "description": "Visitor Management System for the Office of the Head of Civil Service, Ghana",
  "start_url": "/",
  "display": "standalone",
  "background_color": "#1A4D2E",
  "theme_color": "#1A4D2E",
  "icons": [
    { "src": "/icons/icon-192.png", "sizes": "192x192", "type": "image/png" },
    { "src": "/icons/icon-512.png", "sizes": "512x512", "type": "image/png" }
  ]
}
```

### Service Worker

Basic caching strategy:
- **App shell** (HTML, CSS, JS, fonts, logo): Cache-first — serve from cache, update in background
- **API calls**: Network-only — always hit the server, no offline data
- **Images/static assets**: Cache-first with network fallback

Registered in `main.tsx` after app mount.

### Icons

Generate 192x192 and 512x512 PNG icons from the OHCS logo. Store in `/public/icons/`.

### HTML Meta Tags

Add to `index.html`:
```html
<link rel="manifest" href="/manifest.json" />
<meta name="theme-color" content="#1A4D2E" />
<link rel="apple-touch-icon" href="/icons/icon-192.png" />
<meta name="apple-mobile-web-app-capable" content="yes" />
<meta name="apple-mobile-web-app-status-bar-style" content="black-translucent" />
```

### What this enables
- Chrome/Edge "Install" prompt on desktop
- "Add to Home Screen" on mobile
- Opens without browser chrome (no URL bar)
- OHCS icon on desktop/home screen
- Green theme color in mobile status bar

### What this does NOT do
- No offline data access
- No background sync
- No push notifications (Telegram handles this)
